# b4-1 Roadmap

## 목표

순수 HTML, CSS, JavaScript만 사용해 반응형 포트폴리오를 만들고, 이벤트와 상태, DOM 업데이트의 연결을 학습한다.

## 진행 단계

1. 요구사항 분석
   - 필수 섹션, 인터랙션, API 상태, 제출물을 확인했다.

2. 파일 구조 구성
   - `index.html`, `css/style.css`, `js/main.js`, `images/`를 만들었다.

3. HTML 시맨틱 구조 작성
   - Header, Hero, About, Skills, Projects, Contact, Footer를 구성했다.

4. CSS 레이아웃 구현
   - CSS 변수, 다크 모드 변수, Flexbox 네비게이션, Grid 카드 목록, 반응형 브레이크포인트를 적용했다.

5. JavaScript 인터랙션 구현
   - 햄버거 메뉴, 다크 모드, 스크롤 UI, Intersection Observer, 폼 검증을 구현했다.

6. GitHub API 연동
   - `fetch`와 `async/await`로 저장소를 불러오고 로딩, 성공, 에러, 빈 데이터 UI를 분기했다.

7. 문서화
   - 실행과 배포를 위한 `README.md`, 학습을 위한 `b4-1-guide.md`를 작성했다.

## 남은 개인화 작업

- `js/main.js`의 `GITHUB_USERNAME`을 본인 GitHub 아이디로 변경
- `index.html`의 소개 문구, 소셜 링크, Footer 이름 변경
- GitHub Pages 배포 후 `README.md`에 배포 URL과 스크린샷 추가
