# 🚀 미션: 웹 기초 완성, 나만의 포트폴리오 구축

## 1. 미션 개요
외부 라이브러리나 프레임워크 없이 **순수 HTML, CSS, JavaScript**만을 사용하여 반응형 포트폴리오 웹사이트를 구축합니다. 
단순한 UI 구현을 넘어, **`사용자 이벤트 → 상태(State) 변경 → DOM 업데이트`**로 이어지는 웹의 동작 원리를 체득하고 외부 API 연동 및 예외 처리 능력을 기르는 것을 목적으로 합니다. (본 미션은 향후 React/Vue 등 프레임워크 학습의 필수 기반이 됩니다.)

---

## 2. 학습 목표
본 미션을 완료한 후, 다음 사항들을 스스로 설명할 수 있어야 합니다.
*   **HTML**: 시맨틱 태그의 사용 목적과 본인의 구조 설계 기준
*   **CSS**: Flexbox와 Grid의 차이점 및 적절한 사용 상황
*   **JS (DOM)**: `querySelector`를 통한 요소 선택과 `addEventListener`를 활용한 이벤트 제어 흐름
*   **JS (ES6+)**: 화살표 함수, 구조분해 할당, 배열 메서드(`map`, `filter`)의 필요성과 사용법
*   **비동기 통신**: `fetch`와 `async/await`를 활용한 데이터 호출 및 4가지 상태(로딩/성공/에러/빈 데이터)의 UI 표현 방법
*   **상태 관리**: 하나의 기능이 `이벤트 발생 → 상태 변경 → DOM 업데이트`로 연결되는 파이프라인 이해

---

## 3. 개발 환경 및 제약 사항
*   **사용 기술**: 순수 HTML, CSS, JavaScript (Vanilla JS)
*   **사용 금지**: React, Vue, jQuery, Bootstrap, Tailwind CSS 등 일체의 외부 라이브러리/프레임워크
*   **사용 허용**: 아이콘(Font Awesome 등), 웹 폰트(Google Fonts)
*   **코드 컨벤션**:
    *   변수 선언: `var` 사용 금지, `const`와 `let`만 사용
    *   이벤트 바인딩: HTML 인라인 속성(`onclick="..."`) 금지, JS 파일에서 `addEventListener` 사용
    *   스타일링: 인라인 스타일(`style="..."`) 금지, 외부 CSS 파일 분리
    *   JS 파일 연결: `<script defer src="...">` 방식 사용
*   **지원 브라우저**: 최신 Chrome 브라우저 기준 정상 동작
*   **API 주의사항**: GitHub API는 인증 없이 호출 시 **시간당 60회 제한(Rate Limit)**이 있습니다. 403 에러 발생 시 예외 처리(에러 UI 노출)가 반드시 구현되어야 합니다.

## 4. 상세 요구 사항

### 4.1 프로젝트 기본 구성
*   **디렉토리 구조**: 최소한 아래의 역할로 분리되어야 합니다.
    *   `index.html` (메인 페이지)
    *   `css/style.css` (스타일시트)
    *   `js/main.js` (스크립트)
    *   `images/` (이미지 에셋)
*   VS Code + Live Server를 활용하여 실시간 개발 환경을 구성합니다.

### 4.2 HTML 구조 (시맨틱 마크업)
*   무분별한 `div` 사용을 지양하고 시맨틱 태그(`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`)를 사용합니다.
*   **필수 섹션 구성**:
    *   `Hero` (인사말, CTA 버튼)
    *   `About` (자기소개, 프로필 이미지)
    *   `Skills` (기술 스택 목록)
    *   `Projects` (GitHub API 연동 카드 목록)
    *   `Contact` (문의 폼)
    *   `Footer` (저작권, 소셜 링크)
*   **웹 접근성**:
    *   네비게이션 메뉴에 각 섹션으로 이동하는 앵커 링크 적용
    *   모든 이미지에 의미 있는 `alt` 속성 부여
    *   폼 요소에 `<label>`과 `<input>`을 `for` - `id` 속성으로 올바르게 연결

### 4.3 CSS 스타일링 (레이아웃 & 반응형)
*   **CSS 변수 활용**: `:root`에 색상, 폰트, 간격 등을 변수로 정의하여 사용합니다. (다크 모드용 `[data-theme="dark"]` 변수 별도 정의)
*   **레이아웃 기법**:
    *   네비게이션: **Flexbox** 활용 (로고 좌측, 메뉴 우측 배치)
    *   Projects 카드: **Grid** 활용 (`auto-fit`, `minmax`를 사용한 반응형 그리드)
*   **반응형 웹 (Mobile First)**:
    *   브레이크포인트 권장: `768px`(태블릿), `1024px`(데스크톱)
    *   모바일 환경에서는 네비게이션 메뉴를 숨기고 햄버거 버튼을 노출합니다.
*   **시각 효과**: 버튼 및 카드 요소에 `hover` 상태의 `transition`과 `box-shadow`를 적용합니다.

### 4.4 핵심 인터랙션 구현 (JS)
1.  **햄버거 메뉴 토글**: 모바일에서 클릭 시 메뉴 노출/숨김 (`classList.toggle('active')` 활용)
2.  **부드러운 스크롤**: 네비게이션 링크 클릭 시 해당 섹션으로 부드럽게 이동 (`scroll-behavior: smooth` 또는 JS 활용)
3.  **스크롤 탑 버튼**: 스크롤 `300px` 이상 시 노출, 클릭 시 최상단 이동
4.  **네비게이션 스타일 변경**: 스크롤 `60px` 이상 시 네비게이션 배경색 변경
5.  **다크 모드 토글**: 버튼 클릭 시 테마 전환 및 `localStorage`를 활용한 상태 유지(새로고침 시에도 유지)
6.  **스크롤 애니메이션**: `Intersection Observer`를 활용하여 요소가 화면에 보일 때 나타나는 애니메이션 구현 (임계값 `threshold: 0.2` 권장)
7.  **폼 유효성 검사 (Contact)**: 
    *   이름, 이메일, 메시지 필수값 검증 (빈 필드 제출 불가)
    *   이메일 형식 검증
    *   제출 시 `event.preventDefault()`로 새로고침 방지 후, 성공 메시지 또는 에러 메시지 UI 노출

### 4.5 비동기 처리 & GitHub API 연동
*   **API 엔드포인트**: `https://api.github.com/users/{본인아이디}/repos`
*   `fetch`와 `async/await`를 사용하여 데이터를 호출하고 렌더링합니다. (`try/catch`로 예외 처리)
*   **ES6+ 활용**: `map`을 사용하여 데이터를 HTML 문자열(템플릿 리터럴)로 변환하고 화면에 출력합니다.
*   **4가지 상태 UI 필수 구현**:
    1.  **로딩 상태**: 스피너 또는 "로딩 중..." 텍스트 노출
    2.  **성공 상태**: 데이터 기반 프로젝트 카드 리스트 렌더링
    3.  **에러 상태**: "프로젝트를 불러올 수 없습니다" 메시지 및 [재시도] 버튼 노출
    4.  **빈 데이터 상태**: "표시할 프로젝트가 없습니다" 메시지 노출

### 4.6 상태 관리 패턴 구현
다음 중 **3가지 이상의 "사용자 이벤트 → 상태 변경 → 화면 업데이트" 흐름**이 코드에 명확히 드러나야 합니다.
*   *예시 1:* 다크 모드 토글 클릭 → 테마 상태(State) 변경 → `<body>` 클래스 및 전체 스타일 업데이트
*   *예시 2:* API 데이터 요청 → 로딩/성공/에러 상태(State) 변경 → Projects 섹션 DOM 업데이트
*   *예시 3:* 폼 입력 → 유효성 상태(State) 검증 → 에러 메시지 DOM 표시/숨김
*   *예시 4:* (보너스) 필터 버튼 클릭 → 필터 상태(State) 변경 → 목록 재렌더링

### 4.7 배포 및 제출
*   **배포**: GitHub Pages를 이용하여 배포합니다.
*   **README 작성**: 프로젝트 설명, 사용 기술, 배포 URL, 주요 기능 스크린샷(데스크톱/모바일/다크모드)을 포함해야 합니다.
*   **제출물**: GitHub 저장소 URL, 배포된 사이트 URL

## 5. 보너스 과제 (선택 사항)
*   **프로젝트 필터링**: `Array.prototype.filter()`를 활용하여 언어별로 프로젝트를 필터링하는 버튼 구현
*   **타이핑 효과**: Hero 섹션의 텍스트가 타자기처럼 한 글자씩 나타나는 효과 구현
*   **실제 폼 전송**: Formspree 또는 EmailJS를 연동하여 Contact 폼의 데이터를 실제 이메일로 수신
*   **시스템 테마 감지**: CSS 미디어 쿼리(`prefers-color-scheme`)를 사용하여 사용자 OS의 다크 모드 설정 자동 감지 및 적용
